**Red Eye** is a blazing fast, highly asynchronous reconnaissance and OSINT framework designed for red teamers, bug bounty hunters, and cybersecurity researchers. Built with raw speed and stealth in mind, it unleashes 11 simultaneous scanners against a target domain to extract critical infrastructure data, hidden endpoints, and leaked secrets.

## 🚀 Features
* **Asynchronous Engine:** Powered by `asyncio` and `aiohttp`. Scans that usually take minutes are executed in seconds.
* **Ghost Mode (`-q` & `-d`):** Tactical delays, randomized jitter, and dynamic User-Agent rotation to bypass strict WAFs.
* **11 Core Modules:** Subdomains, WAF Detection, GitHub Dorking, Tech Stack, Ports, SSL/TLS, Directories, DNS/SPF, Emails, Wayback Archives, and Subdomain Takeovers.
* **Custom Wordlists (`-w`):** Feed your own wordlists seamlessly into the directory brute-forcer.
* **Proxy Routing (`-x`):** Funnel all scanner traffic through HTTP/S proxies (e.g., Burp Suite, Tor).

## 🛠️ Installation

```bash
# Clone the repository

cd RedEye

# Install dependencies
pip install -r requirements.txt
````

## 🎯 Usage

**Basic Scan (All Modules, Default Wordlist):**
```bash
python3 redeye.py -t example.com
```

**Stealth Minigun (Specific Modules + Custom Wordlist + Tactical Delay):**
```bash
python3 redeye.py -t example.com -m waf,dir,subdomain -w big_list -q --delay 1.5
```

**Full Deep Scan (65k Ports + Output to File):**
```bash
python3 redeye.py -t [https://example.com](https://example.com) -v -o report.txt
```

### Arguments
| Flag | Name | Description |
|------|------|-------------|
| `-t` | `--target` | Target domain or URL (e.g., `target.com` or `https://target.com`). |
| `-m` | `--modules` | Comma-separated list of specific modules to run. |
| `-w` | `--wordlist` | Name of the wordlist in the `/wordlist` directory (without `.txt`). |
| `-q` | `--quiet` | Stealth mode: Adds random jitter and slows down requests. |
| `-v` | `--verbose` | Deep scan mode: Scans all 65,535 ports and uses extended checks. |
| `-x` | `--proxy` | Route traffic through a proxy (e.g., `http://127.0.0.1:8080`). |
| `-d` | `--delay` | Static delay between requests in seconds (e.g., `2.5`). |
| `-o` | `--output` | Save the complete terminal output to a text file. |
